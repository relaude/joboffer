# Recommended Database Table

US-02 requires tracking the MS Form request separately from the candidate record.

```sql
CREATE TABLE CandidateFormRequests
(
    Id INT IDENTITY(1,1) PRIMARY KEY,

    CandidateId INT NOT NULL,
    CandidateApplicationId INT NOT NULL,

    FormType NVARCHAR(100) NOT NULL,
    FormUrl NVARCHAR(1000) NOT NULL,

    RequestSentDate DATETIME NOT NULL,
    DueDate DATETIME NULL,

    SubmissionReferenceNo NVARCHAR(50) NULL,

    Status NVARCHAR(50) NOT NULL,
    -- Draft
    -- Sent
    -- Awaiting Submission
    -- Submitted
    -- Overdue
    -- Cancelled

    EmailSubject NVARCHAR(500) NULL,
    EmailBody NVARCHAR(MAX) NULL,

    Reminder1SentDate DATETIME NULL,
    Reminder2SentDate DATETIME NULL,

    SubmittedDate DATETIME NULL,

    CreatedAt DATETIME NOT NULL,
    CreatedBy INT NOT NULL
);
```

# Page Actions

### Send Request

1. Validate Candidate Email.
2. Generate Submission Reference No.
3. Create CandidateFormRequest record.
4. Send Email with MS Forms Link.
5. Update Candidate Application Status:

```text
Awaiting Candidate Submission
```

### Candidate Submission

When MS Form is completed:

```text
Awaiting Candidate Submission
        ↓
Submitted
```

### Reminder Job

Daily scheduled process:

```text
If Status = Awaiting Candidate Submission
and DueDate <= Today

→ Send Reminder Email
→ Mark as Overdue
```

This design aligns directly with **US-02: Talent Acquisition Partner sends the MS Forms request to the candidate**, while using your existing **Candidates** and **CandidateApplications** tables as the parent records.
