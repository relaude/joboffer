After reviewing the **Process Flow** for **Negotiation and Re-Proposal** (Row 14), this page is significantly different from the Candidate Discussion page.

This stage occurs when:

* Candidate rejects all approved offers
* Candidate requests a higher salary
* TA Partner decides to create a revised proposal
* Existing approved offers must be preserved as historical versions
* A new proposal cycle is created under the same Job Offer transaction

The workflow explicitly states:

> System preserves historical records:
>
> 1. Rejected proposals
> 2. Declined approved offers
> 3. Candidate feedback
> 4. Approver comments
> 5. Prior versions
> 6. Salary band flags and candidate responses
>
> System creates new proposal version number
>
> System routes new proposal set for approval.

### Recommended Database Design

For this screen, do **not overwrite existing offers**. Instead create:

```sql
JobOffer
(
    Id,
    JobOfferNo,
    CurrentVersionNo
)

JobOfferVersion
(
    Id,
    JobOfferId,
    VersionNo,
    Status,
    CreatedBy,
    CreatedDate
)

JobOfferProposal
(
    Id,
    JobOfferVersionId,
    OptionNo,
    ProposedSalary,
    Justification
)
```

This structure directly supports the workflow requirement to maintain **historical proposal versions**, approval history, candidate responses, and negotiation cycles while creating new proposal versions under the same Job Offer transaction.
