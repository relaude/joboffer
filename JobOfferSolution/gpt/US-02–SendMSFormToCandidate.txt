# 1. Purpose

The purpose of this module is to allow the Talent Acquisition (TA) Partner to send a Microsoft Forms link to a candidate for completion of Compensation & Benefits information and submission of required supporting documents.

The system shall record the request, track the submission status, and send reminders when necessary.

---

# 2. User Story

**As a Talent Acquisition Partner**

I want to send a Microsoft Forms request to a candidate

So that the candidate can provide Compensation & Benefits information and upload the required documents.

---

# 3. Actors

| Actor                      | Description                                          |
| -------------------------- | ---------------------------------------------------- |
| Talent Acquisition Partner | Sends the MS Form request to the candidate           |
| Candidate                  | Completes the MS Form and uploads required documents |
| System                     | Tracks request status and sends reminders            |

---

# 4. Preconditions

The following conditions must be met before sending an MS Form request:

1. Candidate record exists.
2. Candidate Application record exists.
3. Candidate has a valid email address.
4. Candidate Application Status is approved for Compensation & Benefits data collection.

---

# 5. Post Conditions

After successful sending of the request:

1. Email is sent to candidate.
2. Request record is created.
3. Status becomes:

```text
Awaiting Candidate Submission
```

4. Audit log is created.

# 7. Page Actions

## Preview Email

Allows the TA Partner to preview the email before sending.

### Validation

* Candidate Email is required.
* Subject is required.
* Email Body is required.

---

## Send Request

Sends the email to the candidate.

### Processing Logic

1. Validate required fields.
2. Generate Submission Reference Number.
3. Save request record.
4. Send email notification.
5. Update Candidate Application Status.

### Status Update

```text
Awaiting Candidate Submission
```

---

## Cancel

Returns user to Candidate List page.

No data is saved.

# 9. Business Rules

## BR-001

Candidate Email Address is mandatory.

---

## BR-002

Only active Candidate Applications may receive a request.

---

## BR-003

Submission Reference Number shall be generated automatically.

### Format

```text
CBF-YYYYMMDD-XXXXX
```

Example:

```text
CBF-20260611-00001
```

---

## BR-004

The system shall store the email content sent to the candidate.

---

## BR-005

The system shall track reminder emails.

---

## BR-006

The system shall prevent duplicate active requests.

Active statuses:

```text
Draft
Sent
Awaiting Candidate Submission
Overdue
```

# 11. Notifications

## Initial Email

Sent immediately after clicking Send Request.

### Sample Subject

```text

```

---

## Reminder Email

Automatically sent based on configured reminder settings.

### Conditions

```text
Status = Awaiting Candidate Submission
AND
Current Date > Due Date
```

# 13. Database Design

## CandidateFormRequests

```sql
CREATE TABLE CandidateFormRequests
(
    Id INT IDENTITY(1,1) PRIMARY KEY,

    CandidateId INT NOT NULL,
    CandidateApplicationId INT NOT NULL,

    FormType NVARCHAR(100) NOT NULL,
    FormUrl NVARCHAR(1000) NOT NULL,

    RequestSentDate DATETIME NOT NULL,
    DueDate DATETIME NULL,

    SubmissionReferenceNo NVARCHAR(50) NULL,

    Status NVARCHAR(50) NOT NULL,

    EmailSubject NVARCHAR(500) NULL,
    EmailBody NVARCHAR(MAX) NULL,

    Reminder1SentDate DATETIME NULL,
    Reminder2SentDate DATETIME NULL,

    SubmittedDate DATETIME NULL,

    CreatedAt DATETIME NOT NULL,
    CreatedBy INT NOT NULL
);
```
