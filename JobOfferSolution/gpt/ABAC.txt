Based on:

1. Your **Job Offer User Stories**
2. The complete **Process Flow** (TA Partner → HROD Head → Division Head → President → Candidate Discussion → Re-Proposal) 
3. Your current architecture:

   * ASP.NET Identity (`ApplicationUser`)
   * Azure AD SSO (`OpenIdConnect`)
   * ASP.NET Roles
   * Custom `JobOfferUsers` table
   * Custom claim `JobOfferUserId` in `AccountService` (retrieved from ClaimsPrincipal)

I would recommend moving from **Role-Based Access Control (RBAC)** to a **Hybrid RBAC + ABAC model**.

---

# Why ABAC for Job Offer System?

Your workflow is not purely role-based.

Example:

### HROD Head

Can approve:

```text
Company = Unilab
Salary <= 100,000
```

But cannot approve:

```text
Company = Amherst
```

---

### Division Head

Can approve:

```text
Business Unit = Sales
```

But cannot approve:

```text
Business Unit = Manufacturing
```

---

### President

Can only approve:

```text
Above Band = True
```

This is ABAC.

Access depends on:

```text
User Attributes
Resource Attributes
Action
Context
```

not merely Role.

# ABAC Policies

Most important table.

---

## AuthorizationPolicies

```sql
CREATE TABLE AuthorizationPolicies
(
    Id INT IDENTITY PRIMARY KEY,

    PolicyName NVARCHAR(200),

    PermissionCode NVARCHAR(100),

    Expression NVARCHAR(MAX),

    IsActive BIT
)
```

Example:

### HROD Approval

```text
Permission:
JO.APPROVE

Expression:

Role == "HROD Head"
AND
Resource.CompanyId == User.CompanyId
AND
Resource.ProposedSalary <= User.ApprovalLimit
```

---

### President Approval

```text
Permission:
JO.APPROVE

Expression:

Role == "President"
AND
Resource.AboveBand == true
```

```text
Azure AD Login
       │
       ▼

       │
       ▼

JobOfferUsers
       │
       ▼

Claims Builder
       │

       │
       ▼

Approval Routing Engine
```

For the Job Offer System described in your user stories and process flow, this **Hybrid RBAC + ABAC design** is much better than pure ASP.NET Identity Roles because approval authority depends on:

* Company
* Legal Entity
* Salary Amount
* Above-Band Status
* Approval Level
* Department
* Business Unit

which are dynamic attributes rather than static roles.
